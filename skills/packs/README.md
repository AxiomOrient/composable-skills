# Skill Packs

> 이 폴더의 pack은 runtime skill이 아니다.
> pack은 스킬을 실행하는 이름이 아니라, 어떤 스킬 묶음을 언제 고르면 되는지 설명하는 카탈로그다.

## 카테고리 pack

| 카테고리 | pack | 포함 중심 |
|---|---|---|
| `scout-*` | [`scout-pack.md`](./scout-pack.md) | 범위 확정, 사실 분석, baseline 측정 |
| `ask-*` | [`ask-pack.md`](./ask-pack.md) | 질문 선명화, 분해, 관점 전환, 수리 |
| `check-*` | [`check-pack.md`](./check-pack.md) | 리뷰, 체크리스트, 경계/실패/보안 점검 |
| `debug-*` | [`debug-pack.md`](./debug-pack.md) | 실패 표면 축소, root-cause 진입 |
| `plan-*` | [`plan-pack.md`](./plan-pack.md) | 브리프, 스펙, IA, 기술 설계, 실행 계획 |
| `build-*` | [`build-pack.md`](./build-pack.md) | 코드 구현, 성능 개선, 완료 조건 반복 |
| `test-*` | [`test-pack.md`](./test-pack.md) | 케이스 설계, 공백 점검, 테스트 가드 |
| `tidy-*` | [`tidy-pack.md`](./tidy-pack.md) | 중복/상수/복잡도 분석과 구조 개선 |
| `doc-*` | [`doc-pack.md`](./doc-pack.md) | 문서 작성, 인덱싱, 루트 README 게시 |
| `ship-*` | [`ship-pack.md`](./ship-pack.md) | 릴리즈 검토, hygiene, GO/NO-GO, publish |

## 고르는 기준

- prefix 경계를 먼저 보고 싶으면 카테고리 pack을 본다.
- 실제 업무 흐름 묶음을 고르고 싶으면 도메인 pack을 본다.
- 실행 예시가 필요하면 [`../SKILL-COMBOS.md`](../SKILL-COMBOS.md)를 본다.
- 설치 후 로컬에 sync된 pack 문서를 그대로 entry guide로 써도 된다.
