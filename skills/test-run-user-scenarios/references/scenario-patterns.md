# Scenario Patterns

Use these patterns when building realistic framework scenarios.

## Happy Path
- First-time maintainer trying to choose the right review or release workflow
- Contributor adding docs and needing a doc-focused path
- Engineer fixing a bug and then adding regression protection

## Failure Path
- Ambiguous prompt with no target scope
- Caller chooses a legacy or wrong skill name
- Prompt asks for review, implementation, and release in one unbounded sentence

## Weird but Plausible
- User mixes Korean and English with half-structured inputs
- User pastes contradictory constraints such as "do not change files" and "implement this now"
- User repeats the same skill twice or combines overlapping workflows
